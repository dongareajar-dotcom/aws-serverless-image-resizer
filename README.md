# AWS Serverless Image Resizer

## 📌 Project Overview

The AWS Serverless Image Resizer is an event-driven application that automatically resizes images uploaded to an Amazon S3 bucket. When a user uploads an image, Amazon S3 triggers an AWS Lambda function that processes the image using the Pillow library and stores the resized image in a separate S3 bucket.

This project demonstrates serverless computing, event-driven architecture, AWS Lambda Layers, IAM permissions, and Amazon S3 event notifications.

# 🚀 Architecture Flow

1. User uploads an image to the Source S3 Bucket.
2. Amazon S3 generates an Object Created (PUT) event.
3. AWS Lambda is triggered automatically.
4. Lambda downloads the image from the Source Bucket.
5. Pillow resizes the image.
6. Lambda uploads the resized image to the Output Bucket.
7. Amazon CloudWatch stores logs for monitoring and troubleshooting.

# 🔧 Implementation Steps

## Step 1: Create S3 Buckets

### Source Bucket

Stores original images uploaded by users.

### Output Bucket

Stores resized images generated by Lambda.

---

## Step 2: Create Lambda Function

Runtime:

```text
Python 3.13
```

Function Name:

```text
image-resizer-sahil
```

---

## Step 3: Create Pillow Lambda Layer

Install Pillow in AWS CloudShell:

```bash
mkdir python
pip3 install Pillow -t python/
zip -r pillow-layer.zip python
```

Create a Lambda Layer and attach it to the function.

---

## Step 4: Configure IAM Permissions

Required permissions:

* s3:GetObject
* s3:PutObject
* CloudWatch Logs

---

## Step 5: Configure S3 Trigger

Event Type:

```text
Object Created (PUT)
```

Destination:

```text
AWS Lambda Function
```
